# V1 Architecture (`07f136a52e5bab02e45f90c1b26f849224d87864`)

```mermaid
flowchart LR
  UI["Frontend UI"] --> PRELOAD["apps/ide/preload.js<br/>codemm.threads.*"]
  PRELOAD --> MAIN["apps/ide/main.js<br/>engineCall()"]
  MAIN --> IPC["apps/backend/src/ipcServer.ts"]

  IPC --> POST["threads.postMessage"]
  POST --> CONV["apps/backend/src/services/sessionService.ts<br/>processSessionMessage()"]
  CONV --> DIALOGUE["apps/backend/src/services/dialogueService.ts<br/>runDialogueTurn()"]
  DIALOGUE --> LLM["apps/backend/src/infra/llm/codemmProvider.ts<br/>createCodemmCompletion()"]
  CONV --> MEM["threadDb + threadMessageDb + threadCollectorDb<br/>spec / collector / confidence / commitments"]

  IPC --> GENREQ["threads.generate"]
  GENREQ --> GEN["apps/backend/src/services/sessionService.ts<br/>generateFromSession()"]
  GEN --> PLAN["apps/backend/src/planner/index.ts<br/>deriveProblemPlan()"]
  PLAN --> ORCH["apps/backend/src/generation/index.ts<br/>generateProblemsFromPlan()"]
  ORCH --> SLOT["apps/backend/src/generation/perSlotGenerator.ts<br/>generateSingleProblem()"]
  SLOT --> LLM
  ORCH --> DOCKER["referenceSolutionValidator.ts<br/>runTestStrengthGate.ts"]
  ORCH --> CHECKPOINT["threadDb problems_json / generation_outcomes_json"]
  DOCKER --> ACT["activityDb.create()<br/>threadDb.setActivityId()"]
  ACT --> OUT["RPC response<br/>activityId + problems"]
```

- `apps/ide/preload.js` and `apps/ide/main.js` are a thin IPC bridge; backend entry is `apps/backend/src/ipcServer.ts`.
- Conversation logic is centralized in `apps/backend/src/services/sessionService.ts::processSessionMessage()` and calls `apps/backend/src/services/dialogueService.ts::runDialogueTurn()`.
- Generation logic is also centralized in `apps/backend/src/services/sessionService.ts::generateFromSession()`, which derives the plan and drives `apps/backend/src/generation/index.ts`.
- Each slot is generated by one LLM call in `apps/backend/src/generation/perSlotGenerator.ts::generateSingleProblem()`, then validated in Docker by `apps/backend/src/generation/referenceSolutionValidator.ts`.
- State lives in thread tables plus checkpointed `problems_json` / `generation_outcomes_json`; final output is an `activityDb.create()` write after reference artifacts are discarded.
