# Handoff: 2026-03-02

## Summary

- Implemented the first production slice of the complexity-failure redesign: structured generation diagnostics, V2 generation IPC APIs, slot-level regeneration, and richer provider metadata capture.
- Kept core invariants unchanged (IPC-only engine boundary, Docker-only untrusted execution, no reference artifact persistence).

## Iteration Log

### Iteration 1 (Generation V2 + Diagnostics Foundation)

- Added generation diagnostics types and progress events:
  - `slot_attempt_summary`
  - `slot_failure_diagnostic`
  - `slot_repair_applied`
  (`apps/backend/src/contracts/generationProgress.ts`, `apps/frontend/src/types/generationProgress.ts`, `apps/backend/src/contracts/generationDiagnostics.ts`).
- Extended LLM completion metadata capture (provider/model/finishReason/truncation/token usage where available) across adapters:
  - OpenAI / Anthropic / Gemini / Ollama
  (`apps/backend/src/infra/llm/types.ts`, `apps/backend/src/infra/llm/adapters/*.ts`).
- Wired metadata and diagnostics into slot generation pipeline:
  - emits structured attempt summaries and remediation hints
  - emits repair strategy events per retry path
  (`apps/backend/src/generation/index.ts`, `apps/backend/src/generation/errors.ts`, `apps/backend/src/generation/perSlotGenerator.ts`).
- Added persisted diagnostics extraction utility:
  (`apps/backend/src/generation/diagnostics.ts`).
- Added backend APIs:
  - `threads.generateV2`
  - `threads.regenerateSlot({ threadId, slotIndex, strategy })`
  - `threads.getGenerationDiagnostics({ threadId, runId?, limit? })`
  (`apps/backend/src/ipcServer.ts`, `apps/backend/src/services/sessionService.ts`, `apps/backend/src/database.ts`).
- Added Electron bridge support for new thread APIs:
  (`apps/ide/main.js`, `apps/ide/preload.js`).
- Updated frontend generation flow:
  - prefers `generateV2` when available
  - consumes new diagnostic progress events
  - fetches and surfaces latest failure remediation via `getGenerationDiagnostics` on failure
  (`apps/frontend/src/app/page.tsx`).
- Updated docs:
  - wrapper/API behavior (`docs/FUNCTIONS.md`)
  - diagnostics troubleshooting (`docs/TROUBLESHOOTING.md`)
  - high-level run logging note (`README.md`)
  - generation IPC naming guidance (`CONTRIBUTING.md`).
- Added/updated tests:
  - diagnostics parser unit test
  - progress event ordering update
  - LLM provider tests updated for metadata
  (`apps/backend/test/unit/generation/generationDiagnostics.test.js`, `apps/backend/test/unit/generation/generationProgressEvents.test.js`, `apps/backend/test/unit/infra/llm/codemmProvider.test.js`).

## Known Issues / Next

- `threads.regenerateSlot` currently supports READY/FAILED sessions before an activity is linked; it intentionally rejects already SAVED threads with an activity_id.
- Attempt diagnostics are now persisted and queryable, but remediation UI is still lightweight text hints; next iteration should add explicit user actions (retry same slot / downgrade complexity / narrow topics) as buttons.
