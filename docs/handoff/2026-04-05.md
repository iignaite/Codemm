# Handoff: 2026-04-05

## Summary

- Landed the first production-hardening slice for local generation reliability.
- The backend now routes models per role, splits generation into explicit slot stages, persists stage-level diagnostics, and surfaces those diagnostics in the IDE.
- Added a full repo-specific architectural audit and staged refactor roadmap grounded in the current Electron + IPC + Docker code paths.

## Iteration Log

### Iteration 1 (Per-Role LLM Routing)

- Replaced the single run-scoped LLM snapshot path with a normalized route-plan model.
- Added backend route-plan types and role selection for:
  - `dialogue`
  - `skeleton`
  - `tests`
  - `reference`
  - `repair`
  - `edit`
- Wired Electron main to build and pass `llmRoutePlan` through IPC instead of a single model snapshot.
- Added `window.codemm.llm.getRoutePlan()` so the renderer can inspect the active routing plan safely.

### Iteration 2 (Dialogue Split + Guided Policy Recompute)

- Changed the dialogue flow so the LLM path is parser-only.
- Assistant prose and confirmation prompts are now rendered deterministically in the backend session service.
- Fixed guided fallback state so pedagogy policy is recomputed after spec fallback instead of reusing stale guided state.

### Iteration 3 (Staged Slot Pipeline)

- Added a new staged slot pipeline:
  - `skeleton`
  - `tests`
  - `reference`
  - `validate`
  - `repair`
- Added typed intermediate pipeline contracts and retry policy.
- Added weak-route capability gating so weak Ollama routes fail fast on hard or multi-topic slots instead of thrashing.
- Kept deterministic Docker validation in place and limited repair to the failed artifact path.

### Iteration 4 (Run Diagnostics + IDE Surface)

- Extended generation progress events with:
  - route selection
  - stage start / finish
  - escalation
  - terminal stage failure
- Persisted sanitized route-plan summaries into run metadata.
- Extended generation diagnostics aggregation with:
  - stage timeline
  - route selections
  - LLM vs Docker timing totals
- Added a simple `Run Details` panel on the home screen so failures can be debugged without raw prompt tracing.
- Added routing-profile and per-role model controls in **LLM Settings**, plus weak-route warning copy.

### Iteration 5 (Compatibility + Verification)

- Added a legacy injected-generator seam for backend unit tests so tests that intentionally stub generation can still verify retry logic while production uses the staged pipeline.
- Updated unit tests for the new stage diagnostics and bounded-concurrency strength gate behavior.
- Verified:
  - backend TypeScript build
  - backend unit suite
  - frontend production build

## Known Issues / Next

- Generation cancellation and admission control are still not implemented.
- The route planner supports per-role selection and escalation plumbing, but `Auto` still uses simple local-profile heuristics rather than full catalog-aware optimization.
- Warm reusable judge workers are intentionally deferred until real diagnostics prove Docker cold-start cost is the dominant latency issue.
- The new architectural audit shows additional structural debt:
  - coordinator files are too broad
  - generation still has legacy/new path drift
  - renderer/backend contracts are duplicated
  - many app-local docs still describe the old HTTP/auth architecture
