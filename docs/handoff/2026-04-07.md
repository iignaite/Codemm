# Handoff: 2026-04-07

## Summary

- Replaced the old exception-driven generation flow with a persistent run/slot state machine.
- Generation is now `runId`-scoped end to end and recovers deterministically after crashes.
- Judge execution now returns structured failure categories and uses stricter Docker sandboxing.

## Iteration Log

### Iteration 1 (Persistent Generation State Machine)

- Added persistent generation execution tables:
  - `generation_runs`
  - `generation_slot_runs`
  - `generation_slot_transitions`
- Added shared contracts for:
  - thread terminal generation states
  - generation run status
  - generation slot stages
  - generation slot terminal outcomes
- Refactored thread generation to:
  - create a durable run before slot execution
  - seed durable slot rows
  - persist slot-stage transitions from progress events
  - derive run/thread outcomes from slot results instead of the first thrown exception
- Added startup reconciliation so stale `PENDING` / `RUNNING` generation runs are repaired on engine boot and stuck thread state is rewritten.

### Iteration 2 (Run-Scoped IPC + Frontend Correlation)

- Reworked the generation progress bus to be keyed by `runId`.
- Updated backend thread IPC so:
  - `threads.generate`
  - `threads.generateV2`
  - `threads.subscribeGeneration`
  all accept or preserve `runId`.
- Updated Electron IPC/preload and frontend bridge clients so the UI subscribes to a specific generation run before it starts.
- Updated thread loading logic to surface latest persisted generation run metadata and accept the new terminal thread states.

### Iteration 3 (Judge Hardening + Failure Classification)

- Added structured judge failure categories:
  - `COMPILE_ERROR`
  - `TEST_FAILURE`
  - `EXEC_TIMEOUT`
  - `OUTPUT_LIMIT`
  - `INFRA_ERROR`
- Hardened Docker execution with:
  - explicit container names and forced cleanup
  - output limit detection
  - PID / CPU / memory limits
  - `no-new-privileges`
  - `cap-drop=ALL`
- Switched Java judging to compile into `/tmp/classes` so `/workspace` is mounted read-only.
- Added explicit compile-time and execution-time timeout markers inside judge commands so reference validation no longer collapses everything into a generic timeout.

### Iteration 4 (API Surface Cleanup, Tests, Docs)

- Deprecated fake targeted slot-regeneration strategies at the backend service boundary; only `retry_full_slot` is currently allowed until stage-targeted resume exists.
- Added unit coverage for:
  - run status aggregation
  - run-scoped progress isolation
  - timeout-stage judge classification
- Added integration coverage for stale-run reconciliation and stuck-thread recovery.
- Added `docs/architecture/GENERATION_STATE_MACHINE.md` and updated top-level docs to reflect the new model.

### Iteration 5 (Real-Test Bootstrap Clarification)

- Added a shared real-provider test bootstrap in `apps/backend/test/helpers/loadRealProviderAuth.js`.
- Real backend provider tests now consistently load keys from:
  - process environment
  - `apps/backend/.env`
  - repo-root `.env`
- Verified that the backend real-provider tests still require the key in the backend test process; the Electron `safeStorage` key used by Codemm-Desktop is not automatically available to backend integration tests.
- Verified the real-provider integration suite boots cleanly and skips deterministically when `CODEMM_RUN_REAL_PROVIDER_SMOKE=1` and provider keys are not present.

### Iteration 6 (Security Hardening Follow-Up)

- Added ownership validation for `threads.subscribeGeneration` so a caller-supplied `runId` must belong to the requested thread before replay or live subscription is allowed.
- Added a regression test for the new `runId` authorization check.
- Hardened Docker execution to run judge and `/run` containers as an unprivileged UID/GID instead of the default container root user.
- Updated judge temp-directory permissions so the read-only bind mount remains accessible to the unprivileged container user.
- Added the same `CODEMM_RUN_REAL_PROVIDER_SMOKE=1` gate to the full real-LLM generation E2E suite so plain integration invocations do not silently spend tokens when provider keys are present in `.env`.

### Iteration 7 (Real Provider E2E Rewire)

- Added a new aggregate real-provider generation suite at `apps/backend/test/integration/llm/realActivityGenerationE2e.test.js`.
- The aggregate suite now:
  - runs real activity generation for each available provider
  - runs a real auto-provider fallback flow by masking higher-priority providers and asserting the persisted generation run used the expected fallback provider
  - validates persisted generation state via `generation_runs`, `generation_slot_runs`, and `generation_slot_transitions`
- Added the missing OpenAI real smoke test to `realProviderIntegrations.test.js` so the smoke suite now covers OpenAI, Anthropic, and Gemini consistently.
- Reduced the default aggregate real-E2E matrix to a cost-bounded `java` / `stdout` / `count=1`; the existing `CODEMM_E2E_LANGS`, `CODEMM_E2E_STYLES`, and `CODEMM_E2E_COUNTS` env vars still expand the matrix when needed.

### Iteration 8 (E2E Test Cleanup)

- Collapsed the four duplicated stubbed language activity-generation E2E files into one matrix suite:
  - `apps/backend/test/integration/languages/activityGenerationMatrix.test.js`
- Removed redundant one-file-per-provider real E2E wrappers and kept provider scoping inside the aggregate real suite via `CODEMM_E2E_PROVIDERS`.
- Updated remaining stubbed integration tests to assert the current terminal generation states (`COMPLETED` / persisted run status) instead of the obsolete `SAVED` contract.
- Renamed the hard-intent weak-tests regression description so it matches the current state machine semantics.
- Updated backend test documentation to point at the matrix suites instead of deleted wrapper files.

## How to Run

- Dev: `npm run dev`
- Full build: `npm run build`
- Backend unit tests for services: `npm --workspace codem-backend run test:unit -- services`
- Backend integration tests for DB flows: `npm --workspace codem-backend run test:integration -- database`
- Real provider smoke checks:
  - `CODEMM_RUN_REAL_PROVIDER_SMOKE=1 npm --workspace codem-backend run test:integration -- llm/realProviderIntegrations.test.js`
- Real provider activity-generation E2E across all available providers plus auto fallback:
  - `CODEMM_RUN_REAL_PROVIDER_SMOKE=1 npm --workspace codem-backend run test:integration -- llm/realActivityGenerationE2e.test.js`
- Real provider targeted activity-generation E2E:
  - `CODEMM_RUN_REAL_PROVIDER_SMOKE=1 CODEMM_E2E_PROVIDERS=openai npm --workspace codem-backend run test:integration -- llm/realActivityGenerationE2e.test.js`
- Stubbed language matrix E2E:
  - `npm --workspace codem-backend run test:integration -- languages/activityGenerationMatrix.test.js`

## Known Issues / Next

- Stage-targeted slot resume is still not implemented; unsupported targeted strategy names are now rejected instead of silently rerunning full generation.
- The frontend still renders progress from events during active generation; longer term it should read persisted run/slot records directly for richer recovery and history views.
- Existing diagnostics still dual-write to legacy `runs` / `run_events`; that compatibility layer can be reduced once the new run tables back the UI directly.
